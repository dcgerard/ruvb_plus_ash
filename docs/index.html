alpha = 1

  • First try at \(\alpha = 1\) No methods work well when I fit everything correctly assuming \(\alpha = 1\). I did not run MOUTHWASH and BACKWASH with variance inflation because I hadn’t implemented the variance inflation penalty yet to identify the model. Note that I ran all simulations when \(\pi_0 = 0.1\) and the number of controls was 10.
  • Simulate under alpha = 1 but fit assuming alpha = 0. Variance-scaled versions of BACKWASH and MOUTHWASH still work pretty darn well.
  • Look at the different ways to fit MOUTHWASH and BACKWASH. I simulated under \(\alpha = 1\) but I fit under either \(\alpha = 0\) or \(\alpha = 1\) and using either scale_var = TRUE or scale_var = FALSE. For scale_var = TRUE and \(\alpha = 1\) I used the penalty \(-0.01 / \xi\), where \(\xi\) is the variance inflation parameter. In general, it is always better to have scale_var = TRUE. Surprisingly, though, the methods that fit assuming \(\alpha = 0\) actually performed BETTER. But MOUTHWASH and BACKWASH with \(\alpha = 1\) and scale_var = TRUE performed competitively.
  • Using the correct level of sprop seems to help when we have the penalty.

Subsample MOUTHWASH

  • Subsample MOUTHWASH works almost as well as full MOUTHWASH but is much much faster.

\(t\) vs normal

  • Explore normal uniform and \(t\) and \(\alpha = 1\) In general, using \(\alpha = 1\) works much worse. Using a uniform grid with a normal likelihood works just as well. Using a \(t\) likelihood is overly conservative. CATErr is not improved when using the truncated SVD.
  • I should probably change the grid for uniform
  • Normal with uniform performs equally well to normal with normal. \(t\) is only a little more conservative and the only significant difference is slightly worse AUC at \(n = 6\) and \(\pi_0 = 0.9\). I don’t know why.
  • Lots of qq-plots of \(t\)-statistics when \(n = 6\) and include estimated confounders from SVA.
